# Pilot User and Data Readiness

## Required roles

- UTRMC admin
- UTRMC user/read-only user
- Supervisors
- Residents / PG trainees
- Optional superadmin

## Required pilot data

- Institution / hospital
- Departments
- Programs / training tracks
- Residents
- Supervisors
- Supervisor-resident mapping
- Active rotations / schedule
- Logbook baseline
- Leave workflow baseline

## Onboarding flow

1. Collect resident data.
2. Validate imports and mappings.
3. Import users.
4. Map supervisors.
5. Test first login.
6. Resolve password-reset/login issues.
7. Approve pilot activation.

## Data rules

- No direct DB edits unless documented.
- Corrections must be tracked.
- Seed/demo users stay separate from real users.
- Research workflow remains out of pilot.
- Legacy admin route remains out of pilot.
